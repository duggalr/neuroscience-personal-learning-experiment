# Design Context

## Users

A single user: Rahul, software engineer with a CS background, learning neuroscience foundations during a personal 4-week course (currently Day 2 of 28). This app **replaces Codex CLI** as his primary learning surface.

**Usage context**: Mobile-first — commute, coffee, evenings, between meetings. Desktop responsive secondary. Single-user, local-only, behind Tailscale on his own machine. No production deployment, no auth.

**Posture**: He treats himself as an adult learner. No gamification, no patronizing, no celebration. Wants substance.

## Jobs to be done

- Learn one neuroscience concept at a time, deeply, with a contextual AI tutor
- Ask side questions that the tutor connects back to the syllabus
- Get quizzed on **understanding, not recognition** (cause vs measurement vs explanation, identify-the-level, what's incomplete about this claim)
- Review weak concepts on a spaced schedule (bucket SRS: 1/2/5/10 days)
- Maintain a durable markdown knowledge base that the app updates via propose-update flow
- Track progress through the 28-day syllabus without ever feeling like a "course dashboard"

## Brand Personality

**Three words: calm, disciplined, substantive.**

**Voice/tone**:
- Treats the user like an adult — no praise inflation, no hand-holding
- Substantive over chatty — the tutor sends thoughtful structured turns, not 6 short bubbles
- Brief warmth at the right moments, never sycophantic
- No corporate cheerfulness, no Duolingo-style positivity
- The tutor sounds like a thoughtful human teacher, not a chatbot

**Emotional goals**:
- Focused calm
- Confidence that comes from real understanding
- The opposite of dopamine-driven

## Aesthetic Direction

**Reference feel: Linear / refined modern app.** Sharp, minimalist, precision-engineered. Not editorial/paper-warm. The app feels like a high-quality engineered tool that respects the user's attention.

**Theme**: Light only for v0. No dark mode.

**Typography**: System fonts only, no web fonts.
- UI: `-apple-system, BlinkMacSystemFont, "SF Pro Text", system-ui, sans-serif`
- Headings use the same family, just heavier weight + larger scale
- Character comes from spacing, rhythm, and color tinting — not from typeface novelty
- Modular scale, ~1.25 ratio between steps, fixed `rem` (app UI, not fluid)

**Color palette** (OKLCH, tinted toward accent hue):
- **Surface**: warm off-white, ~oklch(98% 0.005 155) — barely tinted toward green
- **Surface raised**: ~oklch(99.5% 0.003 155) — for cards/inputs on the warm background
- **Ink (primary text)**: ~oklch(20% 0.018 155) — dark with quiet green tint, not pure black
- **Muted text**: ~oklch(48% 0.02 155)
- **Faint text / borders**: ~oklch(85% 0.015 155)
- **Accent**: deep refined green, ~oklch(38% 0.09 155) — quiet but present; used sparingly for primary CTA, current-state markers, and progress
- **Accent on accent**: light text on the accent surface, ~oklch(97% 0.01 155)

**60-30-10 visual weight**:
- 60% warm surface
- 30% ink + muted + borders
- 10% accent (CTAs, current marker, progress ring fill)

**Layout & space**:
- 4pt scale tokens (4, 8, 12, 16, 24, 32, 48, 64)
- Generous but contained whitespace, not editorial-luxurious
- Subtle 1px borders, low-contrast surfaces — Linear-style precision boundaries
- No drop shadows by default; use a single hairline border or a faint surface lift
- Mobile-first; content max-widths kept tight (one column, focused)

**Motion**:
- Fast, smooth, refined
- Exponential ease-out (cubic-bezier ~0.16, 1, 0.3, 1) for transitions
- 150-250ms durations
- No bounce, no elastic, no spring overshoot
- Animate `transform` and `opacity` only

## Anti-references (explicit bans)

- Duolingo or any gamified learning app
- Generic course dashboards (Coursera, Khan Academy, Udemy)
- Notion-style information density
- Mascots, badges, streaks, confetti, celebrations
- Dark mode with glowing neon accents (the AI-typical look)
- Border-left accent stripes on cards or alerts
- Gradient text or gradient buttons
- Glassmorphism, blur effects used decoratively
- Bouncy/elastic/spring motion
- Repeated identical card grids
- Sparklines as decoration
- "Welcome back, [Name]" with name interpolation — just "Welcome back"
- Time-of-day greetings ("Good morning")
- Skip buttons or "mark complete" affordances anywhere

## The Memorable Thing

It's the **integration** of three things experienced together:
1. **Calm aesthetic** — paper-quiet, no chrome, generous space
2. **Structural discipline** — no skip, quiz is the gate, the system enforces rigor
3. **Substantive tutor voice** — feels like a thoughtful teacher, not a chatbot

No single element is the headline. The cohesion is.

## Design Principles (guide every decision)

1. **One clear next action.** Today never offers a menu when one move is correct. Continue picks for you.
2. **Hide complexity until useful.** Concept graph, source log, syllabus exist — they don't dominate the learning moment.
3. **Contextual tutor, not generic chat.** The chat always knows current day, concept, weak concepts, recent notes.
4. **Discipline through structure, not politeness.** The UI itself enforces no-skip — affordances are removed, not just discouraged.
5. **Concept-first, not dashboard-first.** The atomic unit is the concept being learned, not the day, not the metric.
6. **Substantive over chatty.** Rich tutor turns with markdown structure (causal chains, comparisons, code blocks, ASCII diagrams). Not chat-bubble noise.
7. **Soft cap, never silent truncation.** When something is limited (reviews/session), always show the user the cap and let them opt into more.

## Accessibility

- WCAG AA contrast minimum on all text
- Reduced motion respected: disable transitions if `prefers-reduced-motion`
- Touch targets minimum 44×44 on mobile
- Keyboard navigation works for desktop usage
- No reliance on color alone for state (use icons + text alongside green/amber/red)
